# TurboQuant-Family Compression on CLIP Multimodal Embeddings

**CS 639: Introduction to Foundation Models — Final Project, Spring 2026**
Team: aagarwal235, akumar323, cajain, rsingh, sagarwal, sbarjatya, sdatta9

Evaluates QJL, PolarQuant, TurboQuant, and FAISS-PQ compression of 512-dim CLIP ViT-B/32 embeddings on Flickr30k, across bit-widths {1, 2, 4, 32} and four retrieval tasks (cross-modal and same-modal). Central finding: cross-modal retrieval degrades 10–12× faster than same-modal retrieval under compression due to CLIP's modality gap.

## Implementations

### V1 — `tq-c/`

The first working implementation. Functional but deviates from the papers in three ways:

- **QJL**: uses a plain Gaussian projection instead of SRHT; wastes sign bits on low-variance dimensions.
- **PolarQuant**: uses Lloyd-Max codebooks fitted on CLIP data instead of analytic Beta-distribution codebooks.
- **TurboQuant**: uses a Cartesian MSE quantization stage (`turbo_mse.py`) instead of PolarQuant as the main stage.

See `tq-c_report.pdf` for results produced by this version.

```
tq-c/src/

    polarquant.py     Empirical-codebook PolarQuant (V1)
    turboquant.py     MSE-stage TurboQuant (V1)
    turbo_mse.py      Cartesian scalar quantization stage (V1 only)

```

### V2 — `turboquant-clip/`

The paper-faithful revision. All numbers in `final_report.tex` and `figs2/` come from this version. Key changes over V1:

| Component | V1 | V2 |
|-----------|----|----|
| QJL projection | Gaussian | SRHT (whitens non-isotropic CLIP coords) |
| PolarQuant codebooks | Empirical (Lloyd-Max on CLIP data) | Analytic (Beta-distribution PDF, data-free) |
| TurboQuant main stage | Cartesian MSE scalar quantizer | PolarQuant (matches Algorithm 2 of arXiv:2504.19874) |
| TurboQuant residual QJL | Gaussian, no norm scaling | SRHT + norm scaling (`cosine=False`) |
| Seed handling | Shared seed across stages | Independent seeds per stage |

Cross-modal Recall@10 at 2 bits improved: TurboQuant T1 +104% (0.157 → 0.321), T2 +100% (0.263 → 0.525).

## Reproducing the results

### Prerequisites

Embeddings must exist at `turboquant-clip/data/image_embeddings.npy` and `data/text_embeddings.npy`. Generate them once on a GPU (free Colab T4 is sufficient):

```bash
cd turboquant-clip
python -m src.embed encode --out-dir /content/drive/MyDrive/cs639/data
python -m src.embed finalize --out-dir /content/drive/MyDrive/cs639/data
```

Resumable: if the session dies, re-run the same command — chunks are saved atomically.

### Install dependencies

```bash
cd turboquant-clip
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

### Run the full V2 experiment

```bash
python -m src.eval.experiment \
    --data-dir data \
    --out-dir results \
    --seeds 0,1,2,3,4 \
    --max-queries 1000
```

### Generate figures (into `figs2/`)

```bash
python -m src.analysis.plots --results-dir results --out-dir ../figs2
python -m src.analysis.geometry --data-dir data --out-dir results
```

### V1 vs V2 comparison

```bash
python generate_tables.py        # prints recall tables + delta to stdout
python generate_report_pdf.py    # builds report_v1_vs_v2.pdf
```

Expected runtime: ~45 minutes on a modern laptop CPU (all seeds, all methods).

### Compile the final report

```bash
pdflatex final_report.tex
pdflatex final_report.tex   # second pass for cross-references
```

Fill in the two `PASTE_` placeholders (Colab link and results link) and the LLM appendix before submitting.

## Key results summary

Uncompressed baselines (Recall@10): T1 text→image = 0.498, T2 image→text = 0.741, T3 text→text = 0.528, T4 image→image = 1.000.

| Method | Bits | T1 txt→img | T2 img→txt | T3 txt→txt | T4 img→img |
|--------|------|-----------|-----------|-----------|-----------|
| FAISS-PQ | 4 | 99.0% | 99.3% | 99.4% | 99.4% |
| PolarQuant | 4 | 98.4% | 98.1% | 100% | 100% |
| TurboQuant | 4 | 94.6% | 95.7% | 99.6% | 100% |
| FAISS-PQ | 2 | 94.0% | 90.8% | 94.5% | 97.1% |
| PolarQuant | 2 | 81.5% | 84.9% | 98.9% | 99.5% |
| QJL | 1 | 46.8% | 51.9% | 95.3% | 91.6% |

Values are retention (% of uncompressed Recall@10). Cross-modal tasks (T1, T2) degrade far faster than same-modal tasks (T3, T4) at every bit-width and for every compressor — confirming H1.
